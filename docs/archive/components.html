主题声明语法

三种入口等价,一个 zone 块只需要声明一次。优先级 A > C > B。不写就走默认 editorial。

入口 A · magazine-cover 属性(推荐 95% 场景)

杂志封面自带 theme 属性,整块内容继承该主题:

::magazine-cover theme=literary title="梅雨季" ::chapter ... // 自动继承 literary ::text ...

入口 B · 顶格 ::theme(无封面时)

::theme 是伪组件,只影响主题,不会渲染任何 UI:

::theme purple # 备忘便签 正文段落,可以用 **加粗** 强调重点。
markdown 接管系必用 B
purple/sky/pop/sage/note 5 套接管系主题不写 zone 组件,只用 markdown。因为没有 magazine-cover 挂载点,必须走 ::theme 顶格声明。

入口 C · 围栏 meta

整段都是 zone 组件时,把主题放在围栏 meta 上:

```zone theme=data ::chapter ... ::line ... ```

CSS 变量定制

ZoneDSL 所有主题基于同一套 CSS 变量。业务方可以在自己的小程序里 override 这些变量实现品牌色定制——不用改核心代码。

变量用途
--mz-bg页底
--mz-panel内容卡底(纯白通常)
--mz-panel-soft次级卡底
--mz-ink主字
--mz-ink-soft次字
--mz-ink-mute弱字
--mz-line / --mz-line-strong分隔线
--mz-accent主强调色(品牌色核心)
--mz-accent-warm强调块底色
--mz-accent-soft浅强调
--mz-accent-tint微染背景
--mz-gradient-warm封面/hero 渐变
--mz-font主题级默认字体族(v2.2)
--mz-title-tracking主标题字距(v2.2)

业务定制例子——把整站换成品牌粉色:

/* 业务侧 wxss */ .brand-magenta .zone-block { --mz-accent: #E91E63; --mz-accent-warm: #F06292; --mz-accent-soft: #FCE4EC; --mz-bg: #FDF7F9; --mz-gradient-warm: linear-gradient(135deg, #E91E63 0%, #F06292 100%); }

12 主题一览

杂志系 7 套作用于 zone 组件(.zn-*)。markdown 接管系 5 套(v2.8+)让普通 markdown 元素(.h2w__*)也整体切换视觉。

顶部主题切换器可以实时预览——切一次,整站配色/字体/氛围就跟着变。

AI 输出策略
默认不写 ::theme——让内容走默认 editorial。只在用户明确要求特定风格或内容气质冲突时才显式声明主题。
