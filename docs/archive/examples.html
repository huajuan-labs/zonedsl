# ::ZoneDSL 是什么?

**::ZoneDSL 小程序版**是来自*花卷实验室*的

MIXED · MARKDOWN + ZONE

混合渲染 · markdown 与 zone 并存

用户问「详细讲讲 zone-node 组件怎么工作」—— skill 触发,但内容以叙述为主、 结构化点缀。skill 允许 markdown 段落 + zone 组件穿插 —— 前者管叙事、 后者管数据/流程/时间点。这就是 ZoneDSL 「无侵害扩展」的核心场景。

AI 输出 · markdown 段落 + zone 组件穿插
# zone-node 组件是怎么工作的

**zone-node** 是 zone-DSL 的**统一派发组件** ——
不管你写的是 `::text` 还是 `::magazine-cover`,
最终都会被 towxml 解析成同一个 `<zone-node>` 节点,
再由内部的 60+ 个 `wx:elif` 分支按 tag 派发到对应模板。

::callout "三个关键设计:virtualHost 让 host 透明、styleIsolation 跨作用域命中、addGlobalClass 让主题类落到子组件。三合一让 zone-node 变成一个视觉上完全消失的容器。"

## 数据流

从 AI 吐字到最终渲染,整个链路只有三步:

::data-board cols=3
  item "Parse" value="~2ms" desc="DSL → AST"
  item "Transform" value="~1ms" desc="AST → nodes"
  item "Render" value="~8ms" desc="nodes → DOM"

## 为什么不冲突

- markdown 的 `# 标题`、`**加粗**`、`- 列表` 照常走
- zone 语法用 `::` 前缀,和 markdown 的语法字符完全分开
- parser 先扫 zone 块占位,再让 towxml 处理剩下的纯 markdown

zone-node 组件是怎么工作的

zone-node 是 zone-DSL 的统一派发组件 —— 不管你写的是 ::text 还是 ::magazine-cover, 最终都会被 towxml 解析成同一个 <zone-node> 节点, 再由内部的 60+ 个 wx:elif 分支按 tag 派发到对应模板。

三个关键设计:virtualHost 让 host 透明、styleIsolation 跨作用域命中、addGlobalClass 让主题类落到子组件。三合一让 zone-node 变成一个视觉上完全消失的容器。
