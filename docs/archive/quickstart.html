快速上手

本页帮你在 5 分钟内跑通 ZoneDSL 的第一个 demo,并说明如何 把这套引擎迁移到你自己的微信小程序。

核心链路

ZoneDSL 的工作流水线本质上是四步:

  1. AI 后端流式吐出一段 markdown + zone-DSL 混排文本
  2. 前端 agentChat 页把 content 拿到,交给 buildAst()
  3. towxml/index.js 里的 extractZoneBlocks() 先拆出 zone 块, markdown 走 towxml 原有解析,zone 块交给 zone-dsl/toWxNodes.js 转成组件节点树
  4. 页面渲染层遇到 item.isZone 派发给统一的 <zone-node> 组件, wxml 里 60+ 个 wx:elif 分支各自渲染成对应 zone 组件
TL;DR
你只需要产出 DSL 文本,剩下的解析、组件派发、主题作用域、 流式安全全部由 ZoneDSL 引擎在小程序端搞定。

Hello ZoneDSL

最短的一个 zone 组件:

::text "你好, ZoneDSL"

渲染出来就是一段正文文字。::<组件名> 就是 ZoneDSL 的核心语法。

纯 markdown 用法

ZoneDSL 不排斥 markdown——你完全可以只写 markdown,ZoneDSL 引擎会照常 把它交给 towxml 渲染,得到跟你熟悉的 markdown 一样的结果:

# 一级标题

正文段落,可以用 **加粗**、*斜体*、==高亮==、`inline code`。

- 列表项 1
- 列表项 2

| 表头 | 数值 |
| --- | --- |
| A | 1 |

加入 zone 组件

markdown 里穿插 :: 组件(前后各空一行),得到杂志式结构化输出:

# 今日热搜三看点

今天有三条线索都值得看。

::magazine-cover tag="HOT SPOTS · 07.03"
  title="今日三焦点"
  bg=light

## 焦点一: 示例综艺

正文段落...

::numbered-list "看点在哪"
  item "四姐气场碰撞"
  item "南美世界杯氛围"
规则
顶格 :: 前后必须各留一行空行,让 towxml 识别 zone 块边界。子项用 2 空格缩进,不能用 Tab 或 4 空格。

切换主题

ZoneDSL 内置 12 套主题——7 套杂志系作用于 zone 组件, 5 套 markdown 接管系让纯 markdown 也整体变杂志视觉。

不写 ::theme 就是默认 editorial 暖橙风。要切主题:

::theme purple

# 我的备忘录

正文段落——现在整块内容会变成紫底黄胶囊白卡片的杂志视觉。

可用主题:editorial / literary / serious / data / serene / warm / luxe / purple / sky / pop / sage / note。详见 组件全览 · 12 主题。

迁移到你自己的小程序

ZoneDSL 已拆成 npm 包:@zonedsl/core(parser) + @zonedsl/web(Web 渲染) + @zonedsl/skill(AI 规范),npm i 即用。

1. 复制目录

把项目里这一个目录整体拷贝到你自己小程序的分包下(比如 your-package/zone-plugin/):

zone-plugin/
├── towxml/                # markdown 渲染器
├── zone-dsl/              # DSL 解析器
└── zone-components/
    ├── zone-node/         # 统一派发组件
    └── themes/            # 12 主题

2. 修引用路径

在你消费方页面的 index.json 里注册 towxml 组件:

{
  "usingComponents": {
    "towxml": "/your-package/zone-plugin/towxml/towxml"
  }
}

在 JS 里 require:

const towxml = require('../zone-plugin/towxml/index.js')
const ast = towxml(content, 'markdown')
// 把 ast 塞进 <towxml nodes="{{ast}}" /> 里就行

3. 实现 zone-action 分发器

如果 DSL 里用了 ::button intent=xxx,业务方需要在页面里实现分发器:

onZoneAction(e) {
  const { intent, value } = e.detail
  switch (intent) {
    case 'search':
      wx.navigateTo({ url: `/pages/search/index?q=${value}` })
      break
    case 'followup':
      // 填充输入框
      break
    // ... 其他 intent 按你项目情况实现
  }
}
备注
Intent 白名单一共 11 个,详见项目内 zone-plugin/zone-dsl/INTENTS.md。 未识别的 intent 静默降级为纯样式按钮,不会报错。
